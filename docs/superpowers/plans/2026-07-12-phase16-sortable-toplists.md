# Phase 16 — Sortable Toplist Ranked Lists Implementation Plan

> REQUIRED SUB-SKILL: superpowers:subagent-driven-development.

**Goal:** Extend click-to-sort to the remaining tabular displays that lack it — the `Toplist` ranked lists — by adding an opt-in `sortable` mode to the shared `Toplist` component (reusing Phase-15's `useSortableRows`/`compareBy`), and enabling it on the ~18 full ranked lists (cost, DNS, hotspots, pareto, scorecard, movers, slowest, efficiency, ...). Compact teasers + the 2-row RCODE list stay opt-out.

**Constraints:** app-only. Version bump **0.10.0** (this ships together with the merged-but-undeployed v0.9.0 typed-table sorting). Sort RAW `value`/`label` (not formatted `display`). `initial={key:null}` → caller's current value-desc order preserved until the user clicks (zero behavior change when off + before interaction). Bar-width `max` computed over ALL rows regardless of sort order. Strings via t() ko+en; tokens only; aria on sort controls. Preserve `onSelect` row-click, status dots, `sub`, `display`, empty state, all testids. NO Co-Authored-By.
