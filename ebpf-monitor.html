왜 NFM Dashboard인가

흩어진 지표 대신, 플로우 하나의 화면.

서비스 간 지연이나 재전송 증가는 CloudWatch·VPC Flow Logs·개별 파드 로그를 오가며 원인을 찾기 쉽지 않습니다. NFM Dashboard는 eBPF 에이전트가 수집한 플로우 데이터를 토폴로지·비용·SLO·AI 진단 네 축으로 재구성해, 문제가 되는 엣지를 먼저 짚어 줍니다.

01

라이브 토폴로지

Pod·Namespace·노드 단위 130개 노드, 218개 엣지를 실시간 그래프로. 문제 엣지는 색으로 즉시 드러납니다.

02

비용 & SLO 가시성

Cross-AZ·Cross-VPC 트래픽의 예상 비용을 플로우 단위로 추정하고, 모니터별 SLO 스코어카드로 가용성을 추적합니다.

03

AI 네트워크 진단

현재 토폴로지·수집 상태·이상 징후를 한 번에 분석해, 어느 엣지·파드가 근본 원인인지 한국어로 요약해 줍니다.

기능

개요부터 진단까지, 빠짐없이.

개요 KPI, 토폴로지, 네트워크 분석, 인사이트, 모니터, 알림, AI 진단 — 하나의 대시보드 안에서 이어집니다.

개요 KPI

데이터 전송량·재전송·타임아웃·RTT(p50/p95)를 트렌드 스파크라인과 함께 한눈에.

전송량 · 재전송 · 타임아웃 · RTT

토폴로지 그래프 & 매트릭스

네임스페이스·클러스터 단위 집계, 지표 전환(전송량/재전송/타임아웃/RTT), 소스→대상 매트릭스 뷰.

React Flow · LIVE

카테고리별(AZ 간/내부, VPC 간 등) 예상 전송 비용과 비용 상위 기여 플로우를 추적.

Cross-AZ 비용 추정

런타임 흐름

수집에서 진단까지, 네 단계로.

  1. 수집 · Collect

    EKS 노드와 독립 워크로드의 NFM 에이전트가 커널 레벨 eBPF로 Pod-to-Pod 플로우를 확보하고, AWS Network Flow Monitor가 이를 top-contributor 플로우 · Workload Insights(11개 흐름 범주) · CloudWatch 지표 · Route53/CoreDNS DNS 로그로 노출합니다.

    eBPF AgentAWS NFM · WI 11 categoriesCloudWatch · DNS47 AGENTS
  2. 집계 & 저장 · Aggregate & Store

    EventBridge Scheduler가 5분마다 Collector Lambda를 돌려 이 지표를 DynamoDB 핫 스토어(7일 TTL)에 기록합니다. DynamoDB Stream → archive-transform Lambda → Kinesis Firehose(Parquet)가 TTL 삭제 전에 모든 플로우를 S3로 아카이브하고, Glue·Athena로 카탈로그화해 /history 페이지가 임의 날짜 범위를 조회합니다.

    EventBridge · 5minDynamoDB · 7d TTLFirehose · S3 ParquetGlue · Athena · /history
  3. 시각화 · Visualize

    Next.js 대시보드가 지표를 라이브 토폴로지 그래프, 네트워크 분석 테이블, 비용·SLO 인사이트로 재구성해 한 화면에 보여줍니다.

    Next.js130 NODES · 218 EDGES토폴로지 · 인사이트
  4. 진단 & 알림 · Diagnose & Alert

    Amazon Bedrock ConverseStream과 AgentCore MCP 게이트웨이(SigV4, 27개 도구)가 이상 징후를 종합해 근본 원인 후보를 제시하고, CloudWatch 경보를 SNS로 통지합니다.

데이터 & 포지셔닝

관측 전용, 변경 권한 없음.

NFM Dashboard는 AWS Network Flow Monitor가 수집한 지표를 읽어 시각화하는 관측 대시보드입니다. 이 브로셔의 화면은 워크숍·데모용 EKS 클러스터 데이터이며, 고객 실 데이터가 아닙니다.

  • 읽기 전용 관측

    NFM 에이전트와 대시보드 모두 플로우 지표를 읽어 보여줄 뿐, 네트워크 경로나 리소스를 변경하지 않습니다.

  • 커널 레벨 수집

    eBPF 기반 수집으로 애플리케이션 코드 변경 없이 노드·파드 단위 플로우를 확보합니다.

  • 기존 CloudWatch 연동

    모니터·알람은 CloudWatch로 바로 연결되어, 기존 관측 파이프라인과 함께 사용할 수 있습니다.

  • 워크숍 데이터로 구성

    본 브로셔의 스크린샷은 데모/워크숍 클러스터(ekscluster01, eksworkshop) 기준이며 실제 고객 환경 데이터는 별도입니다.
